上传新的模考试卷

无需编程。准备好试卷 HTML，按下面三步操作即可上线。

📥 上传流程（全程在网页点击，无需命令行）

下面的「①整理试卷」工具会帮你给试卷打好标签，让它出现在正确的科目分类下。

  1. 整理试卷（可选但推荐）：把试卷 HTML 拖到下方工具区，填写标题/科目/时长，点击「生成并下载」。下载到的新文件已经带好标签。
  2. 上传到 GitHub：打开你的仓库页面，进入 exams 文件夹 → 点 Add file → Upload files → 把试卷 HTML 拖进去 → 点 Commit changes。
  3. 自动上线：上传后系统会自动重建模考列表（约 1 分钟）。刷新前台首页即可看到新试卷，无需任何其他操作。

删除试卷同理：在 GitHub 的 exams 文件夹里删除对应 HTML 文件，列表会自动更新。

① 整理试卷（打标签工具）

在你的浏览器本地运行，不会上传任何文件。处理完直接下载到本机。

📄
点击选择 或把试卷 HTML 文件拖到这里

下载后，把这个新文件上传到仓库的 exams 文件夹即可（见上方第 2 步）。

② 进阶：手动给试卷加标签

如果你想直接编辑 HTML，把下面这段粘到试卷 <head> 里，改成你的内容即可。系统会优先读取这些标签。

<meta name="exam:title" content="学术类阅读模考 1"> <meta name="exam:type" content="reading"> <!-- listening/reading/writing/speaking/full --> <meta name="exam:category" content="academic"> <!-- academic / general --> <meta name="exam:duration" content="60"> <!-- 分钟，0=不限时 --> <meta name="exam:description" content="3 篇文章共 40 题。">

不加也没关系——系统会根据文件名和网页标题自动猜测科目与标题。

③ 进阶：让成绩自动记录到「我的成绩」

你的试卷自带批改功能即可正常使用。若希望学生分数自动同步到成绩页，在你的批改脚本算出分数后，加一行：

window.parent.postMessage( { type: "yysd:score", score: 32, total: 40, band: 7 }, "*");

示范卷 sample-academic-reading-1.html 已内置该写法，可参考。
